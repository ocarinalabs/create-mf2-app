## Installation methods

### Method 1: Interactive CLI (Recommended)

The easiest way to get started is using our interactive CLI:

<CodeGroup>

```bash npm
npx create-mf2-app@latest
```

```bash yarn
yarn create mf2-app
```

```bash pnpm
pnpm create mf2-app@latest
```

```bash bun
bunx create-mf2-app@latest
```

</CodeGroup>

The CLI will guide you through:
1. Naming your project
2. Choosing a template (Landing Page or SaaS)
3. Initializing a git repository

### Method 2: Direct with options

Skip the interactive prompts by providing options directly:

```bash
npx create-mf2-app@latest my-app --template saas --git
```

**Available options:**
- `--template <type>`: Choose `landing` or `saas`
- `--git`: Initialize a git repository
- `--no-git`: Skip git initialization

### Method 3: Clone from GitHub

For maximum control, clone the repository directly:

```bash
# Clone the repository
git clone https://github.com/korrect-ai/create-mf2-app.git

# Navigate to your chosen template
cd create-mf2-app/cli/templates/base-fullstack # or base-frontend

# Install dependencies
npm install

# Start developing
npm run dev
```

## Post-installation setup

### For Landing Page template

The landing page template works out of the box:

```bash
cd your-app-name
npm run dev
```

Your site is now running at [http://localhost:3000](http://localhost:3000).

### For SaaS template

The SaaS template requires additional setup:

<AccordionGroup>
  <Accordion title="1. Set up environment variables" icon="key">
    Copy the example environment file:
    ```bash
    cp .env.example .env.local
    ```
    
    You'll need to add API keys for:
    - **Convex**: Database and backend functions
    - **Clerk**: User authentication
    - **Polar**: Subscription payments
    - **Resend**: Transactional emails
    - **PostHog**: Analytics (optional)
  </Accordion>
  
  <Accordion title="2. Start the development servers" icon="play">
    Run both Next.js and Convex:
    
    **Terminal 1:**
    ```bash
    npm run dev
    ```
    
    **Terminal 2:**
    ```bash
    npx convex dev
    ```
  </Accordion>
  
  <Accordion title="3. Set up your database" icon="database">
    Convex will automatically:
    - Create your database tables
    - Set up real-time subscriptions
    - Deploy your backend functions
    
    No manual migration needed!
  </Accordion>
</AccordionGroup>

## Verifying your installation

Run these commands to ensure everything is set up correctly:

```bash
# Check Next.js is working
curl http://localhost:3000

# Check TypeScript compilation
npm run typecheck

# Check linting
npm run lint

# Build for production (optional)
npm run build
```

<Tip>
  If you encounter any errors, check our [troubleshooting guide](#common-installation-issues) below.
</Tip>

## Project structure overview

After installation, your project will have this structure:

```
my-app/
├── src/
│   ├── app/           # Next.js App Router pages
│   ├── components/    # React components
│   ├── hooks/         # Custom React hooks
│   └── lib/           # Utility functions
├── convex/            # Backend functions (SaaS only)
├── public/            # Static assets
├── .env.example       # Environment variables template
├── package.json       # Dependencies and scripts
├── tailwind.config.ts # Tailwind configuration
└── tsconfig.json      # TypeScript configuration
```

<Note>
  For a detailed explanation of each directory and file, see our [project structure guide](/project-structure).
</Note>

## Common installation issues

<AccordionGroup>
  <Accordion title="Permission denied errors" icon="lock">
    On macOS/Linux, you might need to use sudo:
    ```bash
    sudo npx create-mf2-app@latest
    ```
    
    Or fix npm permissions:
    ```bash
    npm config set prefix ~/.npm
    export PATH="$HOME/.npm/bin:$PATH"
    ```
  </Accordion>
  
  <Accordion title="Network timeout errors" icon="wifi">
    If you're behind a corporate proxy:
    ```bash
    npm config set proxy http://proxy.company.com:8080
    npm config set https-proxy http://proxy.company.com:8080
    ```
  </Accordion>
  
  <Accordion title="Module not found errors" icon="folder-x">
    Clear your npm cache and reinstall:
    ```bash
    npm cache clean --force
    rm -rf node_modules package-lock.json
    npm install
    ```
  </Accordion>
  
  <Accordion title="Port already in use" icon="plug">
    Kill the process using port 3000:
    ```bash
    # On macOS/Linux
    lsof -ti:3000 | xargs kill -9
    
    # Or use a different port
    PORT=3001 npm run dev
    ```
  </Accordion>
</AccordionGroup>
