# Quick Start

**For devs who want to ship fast.** Get your MF2 app running in 5 minutes with the stack AI moves fast with.

### Prerequisites

<CardGroup cols={2}>
  <Card title="Node.js 18+" icon="node">
    MF2 requires Node.js version 18 or higher
  </Card>
  <Card title="Package manager" icon="box">
    npm, yarn, pnpm, or bun installed
  </Card>
</CardGroup>

### Step 1: Run the installer (30 seconds)

<CodeGroup>

```bash npm
npx create-mf2-app@latest my-app
```

```bash yarn
yarn create mf2-app my-app
```

```bash pnpm
pnpm create mf2-app@latest my-app
```

```bash bun
bunx create-mf2-app@latest my-app
```

</CodeGroup>

<Tip>
  **Pro tip**: Add your project name instead of `my-app`. The installer uses the command from the website: `npx create-mf2-app@latest ultrathink`
</Tip>

### Step 2: Choose your template

The CLI will ask you to choose between two production-ready templates:

<AccordionGroup>
  <Accordion icon="window" title="Landing Page">
    Perfect for marketing sites, portfolios, and simple web apps.
    
    **Includes:**
    - Beautiful landing page components
    - Pricing section with tiers
    - FAQ section
    - Contact forms
    - SEO optimized
    - Dark mode support
  </Accordion>
  <Accordion icon="credit-card" title="SaaS Starter">
    Full-stack SaaS application with everything configured.
    
    **Includes everything from Landing Page plus:**
    - User authentication (Clerk)
    - Subscription payments (Polar)
    - Real-time database (Convex)
    - Transactional emails (Resend)
    - User dashboard
    - Admin panel
  </Accordion>
</AccordionGroup>

### Step 3: Start developing

Navigate to your project and start the development server:

```bash
cd my-app
npm run dev
```

<Tip>
  If you chose the SaaS template, you'll also need to run `npx convex dev` in a separate terminal for the backend.
</Tip>

Your app is now running at [http://localhost:3000](http://localhost:3000) üéâ

## What's next?

### Configure your environment

If you chose the SaaS template, you'll need to set up your environment variables:

1. Copy `.env.example` to `.env.local`
2. Add your API keys for:
   - Clerk (authentication)
   - Convex (database)
   - Polar (payments)
   - Resend (emails)

<Note>
  Don't have API keys yet? Check our [environment variables guide](/deployment/environment-variables) for step-by-step instructions.
</Note>

## Deploy to production

When you're ready to go live:

<AccordionGroup>
  <Accordion icon="github" title="1. Push to GitHub">
    ```bash
    git add .
    git commit -m "Initial commit"
    git push origin main
    ```
  </Accordion>
  <Accordion icon="vercel" title="2. Deploy with Vercel">
    1. Go to [vercel.com](https://vercel.com)
    2. Import your GitHub repository
    3. Add your environment variables
    4. Click "Deploy"
  </Accordion>
  <Accordion icon="convex" title="3. Deploy Convex (SaaS only)">
    ```bash
    npx convex deploy
    ```
  </Accordion>
</AccordionGroup>

## Common commands

Here are the most useful commands for developing with MF2:

```bash
# Development
npm run dev          # Start Next.js dev server
npx convex dev      # Start Convex dev server (SaaS only)

# Building
npm run build       # Build for production
npm run start       # Start production server

# Code quality
npm run lint        # Run ESLint
npm run typecheck   # Run TypeScript compiler

# Database (SaaS only)
npx convex deploy   # Deploy Convex functions
npx convex logs     # View function logs
```

## Troubleshooting

<AccordionGroup>
  <Accordion icon="circle-exclamation" title="Port 3000 is already in use">
    Kill the process using port 3000 or run on a different port:
    ```bash
    PORT=3001 npm run dev
    ```
  </Accordion>
  <Accordion icon="key" title="Missing environment variables">
    Make sure you've copied `.env.example` to `.env.local` and filled in all required values.
  </Accordion>
  <Accordion icon="database" title="Convex connection errors">
    Ensure you're running `npx convex dev` in a separate terminal window.
  </Accordion>
</AccordionGroup>
